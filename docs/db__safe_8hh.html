Detailed Description

Difference bounds <=c and <c for DBMs.

Note
This implementation is slightly slower than db_unsafe.hh, but it checks for overflow/underflow, it is portable, and conforms to standards

Build a difference bound.

Parameters
cmp: a comparator
value: a value
Precondition
MIN_VALUE <= value <= MAX_VALUE
Returns
<value if cmp is LT and <=value if cmp is LE
Exceptions
std::invalid_argument: if value is not between MIN_VALUE and MAX_VALUE
Parameters
cmp: a comparator
value: a value
Precondition
MIN_VALUE <= value <= MAX_VALUE
Returns
<value if cmp is LT and <=value if cmp is LE
Note
it is not checked whether value is between MIN_VALUE and MAX_VALUE, hence overflow/underflow may occur
the implementation relies on the left-shift operator which is unspecified when value is negative. Hence, semantics is implementation-dependent. In particular, this is incorrect on architectures that do not use two-complement representation of signed integers (checked by static_assert)

Sum of difference bounds.

Parameters
db1: a difference bound
db2: a difference bound
Precondition
db1 + db2 can be represented
Returns
db1 + db2, i.e. the difference bound #c where c is the sum of values in db1 and db2, and # is LT is at least one of db1 and db2 is LT, and # is LE otherwise
Exceptions
std::invalid_argument: if the sum cannot be represented as a tchecker::dbm::db_t
std::overflow_error: if the sum produces an integer overflow
std::underfow_error: if the sum produces an integer underflow
